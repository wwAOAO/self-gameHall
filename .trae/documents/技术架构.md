## 5. 数据模型

### 5.1 核心数据模型

```
Cell 类型
{
  row: number            // 行索引
  col: number            // 列索引
  isMine: boolean        // 是否是地雷
  isRevealed: boolean    // 是否已揭开
  isFlagged: boolean     // 是否已标旗
  adjacentMines: number  // 周围地雷数量（0-8）
}

GameState 类型
{
  board: Cell[][]        // 二维数组表示的棋盘
  rows: number           // 行数
  cols: number           // 列数
  totalMines: number     // 总地雷数
  flagCount: number      // 已标记旗帜数
  revealedCount: number  // 已揭开格子数
  status: 'idle' | 'playing' | 'won' | 'lost'
  difficulty: Difficulty // 当前难度
  timer: number          // 已用秒数
}

Difficulty 类型
{
  label: string          // 难度名称
  rows: number           // 行数
  cols: number           // 列数
  mines: number          // 地雷数
}
```

### 5.2 常量定义（三种难度）

| 难度 | 行数 | 列数 | 地雷数 |
|------|------|------|--------|
| 初级 | 9 | 9 | 10 |
| 中级 | 16 | 16 | 40 |
| 高级 | 16 | 30 | 99 |

## 6. 核心算法

### 6.1 地雷生成

首次点击后生成地雷，保证点击位置及其周围 8 格安全，使用 Fisher-Yates 洗牌算法随机放置地雷。

### 6.2 数字计算

遍历每个非雷格子，统计其周围 8 格的地雷数量。

### 6.3 自动展开（Flood Fill）

当揭开一个周围地雷数为 0 的空格时，使用 BFS（广度优先搜索）算法自动展开周围所有格子。

### 6.4 胜利判定

所有非雷格子全部揭开（`rows * cols - totalMines` 个格子已揭开）即为胜利。

## 7. Composables 设计

### `useMinesweeper.ts`

核心游戏逻辑 composable，管理所有游戏状态和操作。

**暴露的响应式状态**：
- `board`：响应式棋盘数据
- `gameStatus`：游戏状态
- `timer`：计时器（秒）
- `flagCount`：旗帜计数
- `difficulty`：当前难度

**暴露的方法**：
- `initGame(difficulty?)`：初始化/重新开始游戏
- `handleCellClick(row, col)`：处理左键点击
- `handleCellRightClick(row, col)`：处理右键标记
- `handleChordClick(row, col)`：处理 chord 操作（点击已揭开的数字）
